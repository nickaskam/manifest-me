Financial Analysis

Needs resolution
Issue Unresolved

Baseline Cost Problem — ~$2.50/day with zero users

  • Root cause: API service set to min-instances=1 — one vCPU runs 24/7 even with no traffic
  • Cloud Run always-on cost: ~$2.07/day CPU + ~$0.11/day memory = ~$2.20/day base
  • At current config this is ~$67/month before a single video is generated
  • Options to fix: (1) drop to min-instances=0 and accept ~3s cold starts, or (2) keep warm but use a smaller instance, or (3) move to Cloud Run v2 CPU-always-allocated only during requests
  • Decision needed — cold starts on the API are tolerable (it's just the Django boot, not video generation); the worker already runs at min-instances=0
# Current deploy setting causing the burn:
--min-instances 1   ← this keeps a vCPU running at all times

# Fix (drop in deploy.sh, API section):
--min-instances 0   ← cold start ~2-3s, saves ~$67/month at zero-user stage

veo-3.1-generate-lite ⭐ ~$0.05 ~$0.40 ✅ Preview — test first

Known Issues

Needs attention
Issues

Open Items

  • No StoreKit receipt validation — backend accepts any transaction ID from client; trivially bypassable
  • No rate limiting — registration and manifest endpoints are wide open to abuse
  • DEBUG=True in .env — must be False before App Store launch (exposes stack traces)
  • Hardcoded API URL in iOS code and backend — should be an environment variable or build config
  • Celery still in requirements.txt — bloats Docker image; unused since switching to Cloud Tasks
  • Subscription quota reset happens in-request — a race condition at month boundary could reset at wrong time
  • Face ID toggle — backend doesn't enforce it, only the client does
  • GCS credentials path hardcoded to /app/google_credentials.json in engine.py

Tech Stack

Backend

  • Django 6.0.2 + Django Ninja — API framework
  • SimpleJWT — Token auth
  • Gunicorn — WSGI server (2 workers, 60s timeout on API; 1hr on worker)
  • Neon Serverless PostgreSQL — Database
  • Google Vertex AI (Veo 3.1) — Video generation
  • Google Cloud Storage — Video + template storage
  • Google Cloud Tasks — Async job queue
  • Google Cloud Run — Container hosting (API + Worker)
  • MoviePy 1.0.3 — Video stitching (intro → AI clip → outro)

iOS

  • SwiftUI — UI framework
  • StoreKit 2 — In-app subscriptions
  • LocalAuthentication — Face ID
  • URLSession + Combine — Networking + reactive state
  • Keychain — Secure token storage

Deploy Reference

Deploy to Cloud Run

cd backend

./deploy.sh         # Deploy both API + Worker
./deploy.sh api     # API only (512MB, min 1 instance)
./deploy.sh worker  # Worker only (4GB RAM, 2 CPUs, max 5)
  • Requires gcloud auth login and Artifact Registry access
  • Health check: curl https://manifest-me-api-79704250837.us-central1.run.app/api/health/

Run DB Migrations

cd backend
source venv/bin/activate
export DATABASE_URL="your-neon-connection-string"
python manage.py migrate
